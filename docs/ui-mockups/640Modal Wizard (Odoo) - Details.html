Description

A modal dialog used to capture user input for a specific action, such as providing a mandatory reason for a state change. It overlays the current view, requiring user interaction before they can return to the main application.

Design System Alignment

This component is an organism composed of atomic elements like Action Buttons and Form Inputs. It uses the design system's color palette for backgrounds, text, and states; spacing tokens for padding; and typography for titles and content.

Responsive Specifications

Mobile Adaptations

  • The modal dialog takes up a larger percentage of the screen width.
  • Internal padding is slightly reduced to maximize content space.
  • Footer buttons may stack if there are more than two.

Tablet Adaptations

  • Maintains the desktop layout and sizing.

Desktop Enhancements

  • Optimized max-width for readability.
  • Subtle entrance animation.

Accessibility Features

Keyboard Support

The modal must trap focus, meaning the Tab key cycles only through the modal's interactive elements. The `Esc` key should close the modal.

Screen Reader Support

The dialog uses `role="dialog"` and `aria-modal="true"`. An `aria-labelledby` attribute points to the modal title for context.

Focus Management

When the modal opens, focus is programmatically moved to the first interactive element inside it (e.g., the textarea or close button).

Aria Attributes

  • role="dialog"
  • aria-modal="true"
  • aria-labelledby

Color Contrast

All text, icons, and interactive elements meet WCAG 2.1 AA contrast requirements.
