Description

A layout component used to organize content within a form view into separate, selectable panes. It allows users to switch between different sections of information without leaving the main record.

Design System Alignment

This component utilizes the design system's primary color for active state indication, neutral colors for borders and text, and the standard spacing scale for padding and layout.

Responsive Specifications

Mobile Adaptations

  • Tabs wrap to multiple lines if they exceed the container width.
  • Tab items can grow to fill available space for better touch targets.

Tablet Adaptations

  • Maintains the desktop horizontal layout.

Desktop Enhancements

  • Clean horizontal tab list with clear active state and hover feedback.

Accessibility Features

Keyboard Support

Users can tab between tab buttons and activate them with Enter/Space.

Screen Reader Support

Uses `role="tablist"`, `role="tab"`, and `role="tabpanel"`. ARIA attributes like `aria-selected` and `aria-controls` are used to associate tabs with their panels.

Focus Management

A clear, visible focus ring is applied to the active tab button on keyboard focus.

Aria Attributes

  • role="tablist"
  • role="tab"
  • role="tabpanel"
  • aria-selected
  • aria-controls
  • id

Color Contrast

Active and inactive tab text colors meet WCAG AA contrast ratios against the background.
