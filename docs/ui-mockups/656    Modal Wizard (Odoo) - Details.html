Responsive Specifications

Desktop Layout

  • Modal has an optimized max-width for readability.
  • Subtle entrance animation is used.

Tablet Adaptations

  • Maintains the desktop layout and sizing.

Mobile Adaptations

  • The modal dialog takes up a larger percentage of the screen width.
  • Internal padding is slightly reduced to maximize content space.
  • Footer buttons may stack if there are more than two.

Accessibility Features

Keyboard Navigation

Focus is trapped within the modal. Tab key cycles through interactive elements. 'Esc' key closes the modal.

Screen Reader Support

The dialog uses `role="dialog"` and `aria-modal="true"`. An `aria-labelledby` attribute points to the modal title for context.

Focus Management

When the modal opens, focus is programmatically moved to the first interactive element inside it.

Aria Attributes

  • role="dialog"
  • aria-modal="true"
  • aria-labelledby

Color Contrast

All text, icons, and interactive elements meet WCAG 2.1 AA contrast requirements.
