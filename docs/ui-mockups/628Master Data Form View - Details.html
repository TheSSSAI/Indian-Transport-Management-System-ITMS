Description

A comprehensive form view for creating, reading, and updating master data records (e.g., Vehicle, Driver). This organism is composed of several molecular components including a header with actions/statusbar, smart buttons, data entry fields, notebook tabs for sectioning content, and a chatter/log area for history and communication.

Design System Alignment

This organism is a composition of molecular components (Form Header, Smart Buttons, Notebook Tabs, Chatter) and atomic components (Field Widgets, Buttons, Badges). It heavily relies on the design system's tokens for layout, spacing, typography, and color to create a consistent and coherent user experience aligned with the Odoo framework.

Responsive Specifications

Mobile Adaptations

  • The main two-column grid layout collapses into a single vertical column on screens narrower than 768px.
  • The form header stacks its action buttons and status bar vertically.
  • Field widgets and labels maintain full width for readability and ease of use.

Tablet Adaptations

  • The form may appear as a single column or two columns depending on the viewport width and device orientation, adapting fluidly.

Desktop Enhancements

  • Utilizes a two-column layout to efficiently use screen real estate.
  • Hover states on interactive elements like smart buttons and tabs are fully supported.

Accessibility Features

Keyboard Support

All interactive elements, including buttons, form fields, and tabs, are keyboard navigable and activatable.

Screen Reader Support

Semantic HTML structure is used. All form fields are associated with `

Focus Management

Visible focus rings are applied to all interactive elements on `:focus-visible`.

Aria Attributes

  • aria-label for icon buttons
  • for attribute on labels
  • role='tablist', role='tab', role='tabpanel'
  • aria-selected
  • aria-controls

Color Contrast

All text and UI element colors are chosen to meet a minimum contrast ratio of 4.5:1 (WCAG AA).
