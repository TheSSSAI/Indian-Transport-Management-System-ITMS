Description

An overlay component used to capture mandatory user input for a specific action, such as providing a reason for resuming a trip. It interrupts the user's workflow to gather information before proceeding.

Design System Alignment

This component is an organism composed of atomic components: 'Action Button' for its footer actions and 'Field Widget (Text)' for the input area. It uses design system tokens for colors, spacing, typography, and shadows to maintain visual consistency.

Responsive Specifications

Mobile Adaptations

  • The modal dialog takes up the full width of the screen, with a small margin.
  • The modal body becomes scrollable if its content exceeds the viewport height.

Tablet Adaptations

  • Maintains the desktop appearance with a fixed max-width, centered on the screen.

Desktop Enhancements

  • Centered overlay with a backdrop, providing clear focus on the modal's content and actions.

Accessibility Features

Keyboard Support

The modal should trap focus, meaning the Tab key cycles only through the modal's interactive elements. The Escape key should close the modal.

Screen Reader Support

Uses `role="dialog"` and `aria-modal="true"` to inform screen readers of its purpose. The title is linked via `aria-labelledby`.

Focus Management

Focus is moved to the first interactive element (e.g., the close button or first input) when the modal opens. When it closes, focus returns to the element that triggered it.

Aria Attributes

  • role="dialog"
  • aria-modal="true"
  • aria-labelledby

Color Contrast

All text and interactive elements within the modal meet WCAG AA contrast requirements.
