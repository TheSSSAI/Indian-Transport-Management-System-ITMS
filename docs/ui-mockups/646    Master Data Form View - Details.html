Responsive Specifications

Desktop Layout

  • A two-column grid (`o_group`) is used for the main form fields to optimize space.
  • The form header displays action buttons and the statusbar side-by-side.

Tablet Adaptations

  • The two-column grid may collapse to a single column depending on viewport width and orientation.
  • Maintains desktop-like spacing and component sizes.

Mobile Adaptations

  • The main form grid collapses into a single vertical column on screens narrower than 768px.
  • The form header stacks its action buttons and statusbar vertically to fit narrow viewports.
  • The form sheet loses its padding and rounded corners to provide an edge-to-edge experience.

Accessibility Features

Keyboard Navigation

All interactive elements, including buttons, form fields, and tabs, are keyboard navigable and activatable, following a logical tab order.

Screen Reader Support

Semantic HTML is used (header, main, section, nav). Form fields are correctly associated with labels. ARIA attributes are used for stateful components like tabs (role='tablist', aria-selected).

Focus Management

Visible focus rings are applied to all focusable elements, ensuring clear keyboard navigation.

Aria Attributes

  • role='tablist'
  • role='tab'
  • role='tabpanel'
  • aria-selected
  • aria-controls
  • aria-label

Color Contrast

All text and UI elements meet the WCAG 2.1 AA minimum contrast ratio of 4.5:1.
