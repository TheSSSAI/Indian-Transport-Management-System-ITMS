Component Variations

Variant Name : "Vehicle"

The form view for creating or editing a vehicle record. It includes fields specific to a vehicle such as Truck Number, Model, Capacity, and Ownership Status.

Variant Name : "Driver (inherited)"

This variant extends the base Odoo `hr.employee` form. It adds a 'TMS' tab or section containing driver-specific fields like 'License Number', 'License Expiry Date', and a toggle to mark the employee as a driver. The core employee details (Name, Contact) are inherited.

Variant Name : "Customer (inherited)"

This variant extends the base Odoo `res.partner` form. It adds TMS-specific fields or tabs, such as a list of customer-specific 'Rate Agreements' and potentially multiple 'Shipping Locations'. The core customer details (Name, Address, GSTIN) are inherited.

Variant Name : "Route"

A simpler form for managing pre-defined routes. It contains fields for 'Route Name', 'Source Location', 'Destination Location', and 'Standard Distance'. This form typically does not require complex tabs or a chatter area.

Variant Name : "Material"

A very simple form for managing material types. It would contain a 'Material Name' and possibly a text field for 'Special Handling Instructions'. This is one of the most basic master data forms.
