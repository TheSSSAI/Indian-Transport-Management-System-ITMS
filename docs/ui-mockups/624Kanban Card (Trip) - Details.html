Description

A self-contained card representing a single Trip on a Kanban board. It displays key operational information such as Trip ID, Customer, Route, assigned resources, and its current status using a Status Badge. It includes states for hover and drag-and-drop interactions.

Design System Alignment

Utilizes design system tokens for colors (neutrals, semantic warnings), spacing, typography, and borders. It directly incorporates the atomic 'Status Badge' component.

Responsive Specifications

Mobile Adaptations

  • The card is designed to be fluid and will occupy 100% of its container's width.
  • The parent organism (Kanban board) is responsible for stacking cards vertically on smaller screens, and this component is designed to function correctly in that layout.

Tablet Adaptations

  • Maintains desktop sizing and proportions within the Kanban column structure.

Desktop Enhancements

  • Hover effects provide visual feedback.
  • Visual state for drag-and-drop interaction is defined.

Accessibility Features

Keyboard Support

The entire card is focusable via `tabindex="0"`. The menu button inside is also focusable.

Screen Reader Support

Uses semantic structure with `

`, `
`, and `
`. Key information is readable as plain text. The menu button has an `aria-label`.

Focus Management

A clear, high-contrast focus ring is applied to the card and the menu button on `:focus-visible`.

Aria Attributes

  • aria-labelledby to link the card to its primary identifier (Trip ID).

Color Contrast

The 'on_hold' variant uses a thick left border and a background tint, not just color, to convey its state, ensuring it is accessible to users with color vision deficiencies.
