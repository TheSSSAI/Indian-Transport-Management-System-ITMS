Description

The Trip Form View is a complex organism that serves as the primary interface for creating, viewing, and managing a single trip record. It integrates multiple molecular components like a status header, smart buttons for navigation, tabbed sections for organizing data, and a chatter area for logging and communication. Its appearance and available actions change based on the trip's current state in its lifecycle.

Design System Alignment

This organism is a composition of several molecular and atomic components. It relies heavily on the design system's tokens for layout (spacing, grid), typography, and colors to ensure consistency with the rest of the Odoo backend.

Responsive Specifications

Mobile Adaptations

  • The main two-column grid (`.o_group`) collapses into a single column.
  • The form header (`.form-header`) stacks its action buttons and statusbar vertically.
  • Padding around the main form sheet is reduced to maximize content space.

Tablet Adaptations

  • The layout may either be single-column or two-column depending on the exact width and orientation, thanks to the responsive grid.
  • Maintains desktop-like spacing and component sizes.

Desktop Enhancements

  • Displays a spacious two-column layout for optimal data density and readability.
  • All molecular components (header, smart buttons, tabs) are fully expanded and interactive.

Accessibility Features

Keyboard Support

The entire form is navigable via keyboard. All interactive elements (buttons, links, tabs) are part of the tab order.
