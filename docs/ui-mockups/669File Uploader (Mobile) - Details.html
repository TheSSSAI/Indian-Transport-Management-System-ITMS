Description

A mobile-first file input component designed for touch interfaces. It allows users to select a file from their device (camera or gallery), displays a preview, and shows various states like uploading and error. It is composed of atomic elements like buttons and spinners.

Design System Alignment

This component utilizes the established design system tokens for colors (primary, semantic, neutrals), spacing, typography, and border-radius to ensure a consistent look and feel with the rest of the Driver Portal application.

Responsive Specifications

Mobile Adaptations

  • Component is full-width by default to maximize touch area.
  • Launches native device UI for camera or file selection, ensuring a familiar and accessible user experience.
  • Large, clear text and icons are used for readability on small screens.

Tablet Adaptations

  • The component will scale within its container but maintain its mobile-first design and proportions.

Desktop Enhancements

  • On desktop, this component would likely support drag-and-drop interactions on the dropzone, though this implementation is focused on mobile-touch.
