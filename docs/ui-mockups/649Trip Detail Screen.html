Variant Name: "all_roles"

The Trip Form View is a single component whose appearance and available actions change dynamically based on the trip's current state and the user's role. The examples above demonstrate these variations: a read-only view for an 'In-Transit' trip, and an editable view for a 'Planned' trip. Other states like 'Completed' or 'On Hold' would similarly alter the header buttons and statusbar.
