Responsive Specifications

Desktop Layout

  • Displays all columns side-by-side without scrolling on wide screens.
  • Fixed-width columns ensure readability and consistent layout.
  • Hover effects and other micro-interactions are fully enabled.

Tablet Adaptations

  • Maintains the horizontal layout but enables horizontal scrolling if the screen width is insufficient to display all columns.
  • Touch targets on cards are sufficiently large for interaction.

Mobile Adaptations

  • The board container becomes horizontally scrollable on viewports where all columns do not fit.
  • The primary responsive strategy is horizontal scrolling, not vertical stacking, to maintain the workflow context.
  • A custom scrollbar is used for better aesthetics on mobile.

Accessibility Features

Keyboard Navigation

The board container is horizontally scrollable with arrow keys. Users can Tab between columns and then to individual cards. Cards themselves are focusable.
