Description

Represents a single trip on a Kanban board. It displays key trip information at a glance, such as its ID, route, customer, and status. It is designed to be a draggable element within the 'Trip Kanban Board' organism.

Design System Alignment

This component is built using foundational design system tokens for colors (neutral, semantic), spacing, typography, borders, and shadows. It is composed of the atomic 'Status Badge' component.

Responsive Specifications

Mobile Adaptations

  • The card itself is fluid and will take 100% of its parent column's width.
  • At the organism level (Kanban Board), the columns containing these cards will stack vertically on screens smaller than a defined breakpoint, making the cards appear in a single-column list.

Tablet Adaptations

  • The card maintains its desktop appearance and scales with the width of the Kanban column.
  • Sufficient padding and interactive areas are maintained for touch interactions.

Desktop Enhancements

  • Full hover and drag micro-interactions with shadows and transforms are enabled.
  • Clear focus indicators are provided for keyboard navigation.

Accessibility Features

Keyboard Support

The card is focusable via the Tab key (`tabindex='0'`). A keyboard user can navigate between cards on the Kanban board.

Screen Reader Support

The card is wrapped in an `

` tag. The Trip ID serves as the accessible name via `aria-labelledby`. A concatenated summary of the trip is provided via `aria-describedby` for a better screen reader experience.

Focus Management

A highly visible focus ring is applied on `:focus-visible` to clearly indicate keyboard focus without affecting mouse users.

Aria Attributes

  • tabindex='0'
  • aria-labelledby
  • aria-describedby

Color Contrast

All text-to-background color combinations meet the minimum WCAG AA contrast ratio of 4.5:1.
