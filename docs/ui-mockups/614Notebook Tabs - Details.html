Description

A layout component used to organize content within a form into separate, selectable sections. It allows users to switch between different views of related information without leaving the page, which is essential for complex forms like the Trip or Vehicle master data views.

Design System Alignment

Uses design system tokens for colors (primary for the active state indicator, neutrals for borders and text), spacing (padding in tabs and panels), and typography (font weights and sizes for active vs. inactive tabs).

Responsive Specifications

Mobile Adaptations

  • On smaller screens where horizontal space is limited, the flex container for the tabs will wrap, placing overflow tabs onto a new line.
  • This behavior is consistent with Odoo's standard responsive patterns for notebooks.

Tablet Adaptations

  • The component maintains its horizontal layout, providing more space is available.

Desktop Enhancements

  • The standard horizontal tab layout is the primary design for desktop, offering clear and direct navigation between content sections.

Accessibility Features

Keyboard Support

Users can navigate between tabs using the Tab key. The Enter or Space key can be used to select a focused tab.

Screen Reader Support

Semantic roles (`role="tablist"`, `role="tab"`, `role="tabpanel"`) and ARIA attributes (`aria-selected`, `aria-controls`) link tabs to their corresponding content panels, providing a coherent experience for screen reader users.

Focus Management

A clear, visible focus indicator is applied to the active tab button using `:focus-visible`.

Aria Attributes

  • role="tablist"
  • role="tab"
  • role="tabpanel"
  • aria-selected="true/false"
  • aria-controls="[panel_id]"
  • id="[panel_id]"

Color Contrast

The active tab uses a combination of color, font weight, and a bottom border to indicate its state, ensuring it is not reliant on color alone. Text colors meet WCAG AA contrast ratios.
