Description

A mobile-first organism component that allows a driver to finalize a delivery. It includes fields for the recipient's name and provides a choice between uploading a photo proof of delivery or capturing an e-signature. It's composed of several molecular and atomic components.

Design System Alignment

This organism composes the 'File Uploader', 'Signature Pad', 'Form Input', and 'Large Action Button' components. It utilizes the design system's tokens for layout (spacing), typography (labels, titles), and color (borders, backgrounds, actions) to create a cohesive and consistent user experience within the Driver Portal.

Responsive Specifications

Mobile Adaptations

  • The form uses a single-column layout to be easily usable on screens from 360px upwards.
  • All interactive elements like buttons and form fields are full-width and have large touch targets (min 44px height).
  • The choice between Photo and Signature is presented as a large, tappable segmented control.

Tablet Adaptations

  • The component will scale within the available content width but maintain its single-column mobile-first layout.

Desktop Enhancements

  • This component is optimized for mobile-first use and is not intended for a primary desktop interface.

Accessibility Features

Keyboard Support

All interactive elements, including the segmented control, form inputs, and buttons, are keyboard focusable and can be operated with Enter/Space.
