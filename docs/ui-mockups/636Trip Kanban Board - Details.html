Description

An organism component that visually organizes 'Trip' records into columns based on their status (e.g., Planned, Assigned, In-Transit). It serves as the primary dashboard for managing the trip lifecycle, allowing for drag-and-drop interactions to change a trip's state.

Design System Alignment

This organism is a composition of molecular 'Kanban Card (Trip)' components. It uses the design system's spacing tokens for layout, neutral colors for the background and structure, and typography tokens for column headers.

Responsive Specifications

Mobile Adaptations

  • The main board container becomes horizontally scrollable on viewports narrower than the total width of the columns.
  • Column widths remain fixed to maintain card readability, preventing them from becoming too narrow.

Tablet Adaptations

  • The horizontal scrolling behavior applies to tablet viewports as well, ensuring the layout is consistent and usable across devices.

Desktop Enhancements

  • The board displays as a multi-column layout. If the number of columns exceeds the viewport width, horizontal scrolling is enabled.

Accessibility Features

Keyboard Support

The board container can be scrolled horizontally using keyboard arrows when focused. Each column and card within it is focusable via the Tab key.

Screen Reader Support

The board is structured with semantic HTML. Columns have `aria-labelledby` attributes pointing to their headers. Card counts in headers provide context.

Focus Management

A visible focus ring is applied to all interactive elements, including the cards themselves, ensuring keyboard navigators can track their position.

Aria Attributes

  • aria-labelledby
  • aria-roledescription="kanban board"

Color Contrast

All text and UI elements meet a minimum contrast ratio of 4.5:1 (WCAG AA). The 'On Hold' card variant uses a border and background tint, not just color, to convey its state.
