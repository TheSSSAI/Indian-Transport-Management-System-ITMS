Description

A navigation component typically placed at the top of a form view. It contains a collection of 'smart buttons' that link to related records, often displaying a count of those records.

Design System Alignment

This component uses the design system's spacing, color, typography, and border-radius tokens. It is a composition of icons and text, styled to function as a clear navigational element.

Responsive Specifications

Mobile Adaptations

  • The flex container wraps, allowing buttons to stack onto multiple lines.
  • Buttons can grow to fill available space to create a more uniform look if needed.
  • Gaps between buttons are slightly reduced to save space.

Tablet Adaptations

  • Maintains desktop layout but benefits from the flex-wrap behavior if the container is constrained.

Desktop Enhancements

  • Full hover effects with shadow and transform are enabled.
  • Clear visual distinction between count and label.

Accessibility Features

Keyboard Support

The entire button is a focusable `` element, navigable via the Tab key and activatable with Enter.

Screen Reader Support

Semantic `` tag ensures it's announced as a link. The text content (count + label) provides a clear, accessible name.

Focus Management

A highly visible focus ring is applied using the `:focus-visible` pseudo-class and the `--shadow-focus-ring` token.

Aria Attributes

    Color Contrast

    Text colors (`--color-neutral-800` and `--color-neutral-500`) have sufficient contrast against the white/light-blue background to meet WCAG AA standards.
