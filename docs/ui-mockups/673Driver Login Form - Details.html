Description

A self-contained, mobile-first login form for drivers. It includes fields for email and password, a primary submission button, and provides states for default, submitting, and error feedback. It is optimized for a single-column, portrait mobile view.

Design System Alignment

This organism is composed of 'Form Input (Mobile)' and 'Large Action Button (Mobile)' atomic components. It utilizes the design system's tokens for color, spacing, typography, and border-radius to ensure a consistent look and feel.

Responsive Specifications

Mobile Adaptations

  • The form is designed mobile-first with a single-column layout.
  • Uses a max-width to remain centered and readable on larger mobile screens and tablets.
  • Leverages full-width atomic components (inputs, buttons) for large, easy-to-use touch targets.

Tablet Adaptations

  • The layout remains a single, centered column, providing a consistent experience.

Desktop Enhancements

  • For the showcase, the component is centered on the page. In a real desktop app, it would be part of a larger page layout.

Accessibility Features

Keyboard Support

All interactive elements (inputs, button, link) are focusable and can be operated via keyboard (Tab, Enter/Space). The tab order is logical.
