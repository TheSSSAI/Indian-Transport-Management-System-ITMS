Description

An organism component that provides a visual workflow for managing trips. It organizes 'Kanban Card (Trip)' components into columns representing different stages of the trip lifecycle (e.g., Planned, Assigned, In-Transit). The board is horizontally scrollable on smaller viewports.

Design System Alignment

This organism is a composition of molecular 'Kanban Card (Trip)' components. It utilizes the design system's spacing tokens for layout and gaps, and neutral colors for its background and structure, ensuring a consistent look and feel with the rest of the application.

Responsive Specifications

Mobile Adaptations

  • The main board container becomes horizontally scrollable on viewports where all columns do not fit.
  • A subtle custom scrollbar is applied for better aesthetics.
  • The component does not stack vertically; horizontal scrolling is the primary responsive strategy as per the requirements.

Tablet Adaptations

  • Maintains the same horizontal scrolling behavior as mobile if the screen width is insufficient to display all columns.

Desktop Enhancements

  • Displays all columns side-by-side without scrolling on wide screens.
  • Scrollbar is visible on hover/scroll when needed.

Accessibility Features

Keyboard Support

The board itself can be scrolled horizontally using keyboard arrows when focused. Users can tab between columns and then tab to the cards within each column. The Kanban Card dependency has its own keyboard navigation.

Screen Reader Support

The board is a `

` with an `aria-label`. Each column is also a `
` with `aria-labelledby` pointing to the column title, providing clear context to screen reader users.

Focus Management

Focus is managed by the browser's default behavior. Focus styles on the cards are handled by the molecular component.

Aria Attributes

  • aria-label for the main board
  • aria-labelledby for each column

Color Contrast

All text and background color combinations for the board structure meet WCAG AA contrast ratios.
