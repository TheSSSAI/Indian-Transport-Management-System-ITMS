Description

A molecular component that combines primary form action buttons and a status workflow indicator (statusbar). It serves as the main control header for Odoo form views, allowing users to perform actions and track the lifecycle state of a record.

Design System Alignment

This component is composed of atomic 'Action Button' and 'Status Badge' components. It utilizes the design system's primary and neutral colors for actions and states, and the standard spacing scale for layout and padding.

Responsive Specifications

Mobile Adaptations

  • Flex container wraps to stack action buttons and statusbar vertically on screens < 768px.
  • Statusbar becomes horizontally scrollable if it exceeds the viewport width.

Tablet Adaptations

  • Generally maintains the desktop layout, but actions may wrap if there are many of them.

Desktop Enhancements

  • Side-by-side layout with actions on the left and statusbar on the right.
  • Hover effects on clickable statusbar items.

Accessibility Features

Keyboard Support

Action buttons and clickable statusbar items are keyboard-focusable and can be activated with Enter/Space.

Screen Reader Support

The statusbar is wrapped in a `

` element with an `aria-label` to provide context. The current status has `aria-current="step"` for screen readers.

Focus Management

Visible focus rings are applied to all interactive elements (buttons, statusbar items).

Aria Attributes

  • aria-label="Status workflow"
  • aria-current="step"

Color Contrast

Current status text on background meets WCAG AA contrast ratios. Past/Future states use standard text colors.
