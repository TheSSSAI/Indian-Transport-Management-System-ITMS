Molecular Component: Notebook Tabs

Used to organize content within a form into separate, selectable sections. This component is foundational for complex organism views like Master Data Form View and Trip Form View.

Default State (Tab 1 Active)

This is the default appearance of the component when a form is loaded. The first tab is active by default.

Content for Vehicle Details goes here. This would typically contain form fields for model, capacity, etc.
Content for Documents goes here. This panel would list uploaded compliance documents.
Content for Service History goes here. This panel would show a log of maintenance activities.
Content for Costs goes here. This panel would show profitability calculations and other financial data.

Alternate State (Tab 2 Active)

This demonstrates the visual state when a user selects a different tab. The is-active class moves to the selected tab, and the corresponding panel is shown.

Content for Vehicle Details goes here.
Content for Documents goes here. Note that this panel is now visible.
Content for Service History goes here.
Content for Costs goes here.

Responsive Behavior (Wrapping)

On smaller screens, if the tabs exceed the available width, they will wrap onto the next line. This ensures all tabs remain accessible.

Content for the active tab ('Trip Details') would be displayed here.

Component Variations

Variant Name : horizontal_tabs

The standard and only variant for this component. It arranges tabs in a single horizontal row that wraps onto new lines if space is insufficient. This is the conventional layout for notebook-style navigation within a form.
