Component Variations

Variant Name: Vehicle Form

The primary form for creating and managing Vehicle master data. Includes fields for registration, capacity, ownership, and maintenance schedules. It contains tabs for compliance documents and service history.

Variant Name: Driver Form (Inherited)

Extends the standard Odoo HR Employee form. It adds a 'TMS Driver Information' tab that becomes visible when an employee is flagged as a driver. This tab contains fields for license number, expiry date, and a copy of the license.

Variant Name: Customer Form (Inherited)

Extends the standard Odoo Partner (Contact) form. It adds TMS-specific fields such as a tab for 'Rate Agreements' and another for managing multiple 'Shipping Locations' for the customer.

Variant Name: Route Form

A simple master data form for defining standard transportation routes. It includes fields for a unique Route Name, Source Location, Destination Location, Standard Distance, and Estimated Transit Time.

Variant Name: Material Form

A simple master data form for defining types of materials that can be transported. It includes fields for a unique Material Name and optional 'Special Handling Instructions'.
