Description

A tappable card component that displays a summary of a single trip, including its ID, status, customer, and route. It serves as the primary item in a driver's list of current and past trips, designed for mobile-first interaction.

Design System Alignment

Utilizes the design system's color palette for status tags, typography for information hierarchy, and spacing tokens for internal layout. Adheres to the card component specification for border, shadow, and padding.

Responsive Specifications

Mobile Adaptations

  • Component is designed full-width for mobile viewports (360px+).
  • Vertical stacking of route information ensures readability on narrow screens.
  • Large, tappable area covers the entire card.

Tablet Adaptations

  • Component will retain its full-width mobile layout, fitting within the content column of a tablet view.

Desktop Enhancements

  • On desktop views, cards could be arranged in a grid if desired, but the internal structure remains the same.

Accessibility Features

Keyboard Support

The card is implemented as an `` tag, making it naturally focusable via the Tab key and activatable with Enter.

Screen Reader Support

An `aria-label` provides a comprehensive summary of the trip card's content, ensuring screen reader users get full context without navigating the inner elements.

Focus Management

A clear, high-contrast `:focus-visible` outline is applied, using design system tokens.

Aria Attributes

  • aria-label

Color Contrast

Text colors (`--color-gray-700`, `--color-gray-900`) on white/light gray backgrounds meet WCAG AA contrast requirements. Status tags also meet these requirements.
