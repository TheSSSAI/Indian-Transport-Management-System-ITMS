Description

A comprehensive, mobile-first form for drivers to submit trip-related expenses. It includes fields for expense type, amount, a receipt upload, and conditional fields for diesel-specific data (quantity, odometer). The form is designed for a single-column layout, optimized for touch interaction and quick data entry on the go.

Design System Alignment

This organism is composed of atomic and molecular components from the design system. It uses 'Form Input (Mobile)', 'Dropdown Select (Mobile)', 'File Uploader (Mobile)', and 'Large Action Button (Mobile)'. All spacing, typography, and color are derived from the established design system tokens, ensuring a consistent user experience within the Driver Portal.

Responsive Specifications

Mobile Adaptations

  • The form uses a single-column layout for simplicity on small screens.
  • All form controls and buttons are full-width to maximize touch targets.
  • Font sizes and spacing are optimized for mobile readability.
  • Conditional fields for 'Diesel' expenses appear inline without disrupting the form flow.

Tablet Adaptations

  • The form maintains its single-column layout on tablets, centered within the content area, as it's optimized for a focused, mobile-first workflow.

Desktop Enhancements

  • Not applicable. This component is specifically designed for the mobile-first Driver Portal.

Accessibility Features

Keyboard Support

All form elements, including the file uploader and submit button, are keyboard navigable and activatable.

Screen Reader Support

All form fields are associated with a semantic `

Focus Management

All interactive elements have a clear `:focus-visible` state, ensuring keyboard users can track their position.

Aria Attributes

  • `aria-describedby` would be used to link error messages to inputs.
  • `aria-live='assertive'` would be used for the top-level error banner.

Color Contrast

All text and UI elements meet WCAG 2.1 AA contrast ratios. Error states use both color and text to convey meaning.
